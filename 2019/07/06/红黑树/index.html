红黑树总结

发表于 2019-07-06 | 分类于 数据结构

一颗高度为h的二叉搜索树查找的时间复杂度为O(h)，随着树的高度h原来越大，时间复杂度并不比链表快，因此引入红黑树。

红黑树的性质

  1. 每个节点非红即黑
  2. 根节点是黑色的
  3. 每个叶点（NIL）是黑色的
  4. 如果一个节点是红色的，则他的两个子节点都是黑色的
  5. 对每一个节点，从该节点到其所有后代叶节点的路径上，包含的黑色节点数量相同

红黑树是满足红黑性质的二叉搜索树。

下图是一颗红黑树：

从某个节点出发（不包含该节点）到达一个也节点的任意一条简单路径上的黑色节点个数称为该节点的黑高

一颗红黑树的黑高为其根节点的黑高。

证明：一颗有n个内部节点的红黑树的高度至多为2lg(n+1)。

第一步：先证以任意一个节点x$为根的子树至少包含的内部节点为$$2^{bh(x)}-1$ 个。之中bh(x)为黑高。 如果x的高度为0，则x必为叶节点（T.nil),并且以x为根节点的子树至少包含

个内部节点。考虑一个高度为正值且有两个子节点的内部节点x，每个节点有黑高bh(x)或bh(x)-1，这分别取决于自身的颜色是红还是黑。由于x的子节点的高度比x本身的高度要低，可以得出每个子节点至少有$2^{bh(x)-1}-1$个内部节点。所以，以x为根的子树至少包含

个内部节点。

第二步：设树的高度为h。根据红黑树的性质四，从根到叶子节点（不包括根节点）的任何一条简单路径上都至少有一半的节点为黑色。因此，树高至少为h/2；于是有

把1移到不等式的左边，再对两边取对数，得到$lg(n+1)>=h/2$

所以$h<=2lg(n+1)$ 。 也因此，查询一棵高度为h红黑树的时间复杂度为O(lgn)。

插入

我们可以在O(lgn)时间内完成向一个含有n个节点的红黑树插入一个新节点。插入步骤如下：

  1. 首先将新节点向插入一棵普通二叉搜索树一样插入到树T中。
  2. 将该节点着色为红色。
  3. 判断该节点着为红色后，红黑树的性质能否得到保持。
  4. 如过破坏了红黑树性质第四条（如果一个节点是红色的，则他的两个子节点都是黑色的），则循环将有关节点进行重新着色和旋转。

下图是一个插入红黑树的例子：

  1. 新节点z被插入a树中。

    z首先被着为红色，这么做为了保持性质5（对每一个节点，从该节点到其所有后代叶节点的路径上，包含的黑色节点数量相同），因此，当z插入后，仍然始终满足红黑树的性质1，性质3，和性质5。而性质2是否满足取决于新插入的节点是否为该树的根节点。如果是，则将该节点再次着为黑色。上图a中显然z节点不是根节点，那么，对于5条性质，它破坏了性质4。

  2. 由于z为红色，z的父节点x也为红色。违反了性质4，同时z的叔父节点y也是红色的。所以，重新将x，y着为黑色。如图b，因为x，y着为黑色，所以x，y的父节点z必须着为红色，如果不这么做，z的父节点x到达它的每一个子树的叶子节点的黑高分别为2，3，3，因此，会违反性质5。

  3. 图b仍然违反性质4，因为x的子节点z为红色。同时，x的兄弟节点y为黑色。顾对x，z进行左旋。得到图c。

  4. 图c的x，z节点违反性质4。进行一次又旋，并将新的根节点7着为黑色。
